# Documentation Contract: xiaozhi-esp32 構造解析ドキュメント

## Contract Goal

実装フェーズで追加する `docs/` 配下の技術文書が、開発者向け構造解析資料として最低限満たすべき構成と導線要件を固定する。

## Required Output

- 配置先は `docs/` 配下であること
- 1 つの主文書として完結していること
- 日本語で記述されること
- `xiaozhi-esp32` の全体構造、主要関心領域、代表的主要フローを含むこと

## Required Sections

- 概要
- 対象範囲と対象外
- ディレクトリ構成と責務分担
- 主要関心領域ごとの実装位置
- 代表的主要フロー
- 共通実装とボード固有実装の境界
- 調査時の注意点または未確認事項

## Content Rules

- 少なくとも起動、音声、通信、設定、OTA、ボード差分を扱うこと
- 主要フローは少なくとも起動、通信、音声を含むこと
- 全ファイル網羅や全ボード詳細一覧に拡大しないこと
- 確認済み事項、推定事項、対象外事項を判別できること
- 読者が次に読むべき代表ファイルまたはディレクトリを示すこと

## Navigation Rules

- リポジトリ利用者が追加説明なしに文書を見つけられること
- 既存 README からの導線が必要な場合は、最小限の追加で済ませること

## Validation Rules

- 文書を読んだ開発者が 15 分以内に主要構造を説明できること
- 少なくとも 6 つの関心領域の実装位置を一意にたどれること
- 起動・通信・音声の代表的主要フローを追えること
