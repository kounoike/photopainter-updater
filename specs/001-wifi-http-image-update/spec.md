# Feature Specification: PhotoPainter WiFi HTTP Image Update with ComfyUI Auto-Generation

**Feature Branch**: `001-wifi-http-image-update`
**Created**: 2026-03-29
**Status**: Draft
**Input**: User description: "PhotoPainterの画像更新をWiFi-HTTP経由で行うファームウェア＆配信サーバ＆ComfyUIで自動生成する仕組み"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - デバイスが自動的に最新画像を取得・表示する (Priority: P1)

PhotoPainter デバイスが定期的に配信サーバへ接続し、最新の画像を HTTP 経由で取得して表示を更新する。ユーザーは何も操作せずとも、ディスプレイが自動的に更新されていく体験を得られる。

**Why this priority**: システム全体の中核となる価値提供であり、これが動作しなければ他の機能に意味がない。

**Independent Test**: デバイスを電源投入後、設定した時間内に配信サーバから画像を取得してディスプレイに表示されることを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** デバイスが WiFi ネットワークに接続済みの状態で、 **When** 設定した更新間隔が経過したとき、 **Then** デバイスは配信サーバへ接続して最新画像を取得し、ディスプレイを更新する。
2. **Given** デバイスが配信サーバへ接続を試みたとき、 **When** サーバが利用可能で新しい画像が存在する場合、 **Then** デバイスはその画像をダウンロードし、ディスプレイに表示する。
3. **Given** デバイスがサーバへ接続を試みたとき、 **When** ネットワークエラーや接続失敗が発生した場合、 **Then** デバイスは現在表示中の画像を維持し、次の更新サイクルで再試行する。

---

### User Story 2 - ComfyUI が画像を自動生成して配信サーバへ配置する (Priority: P2)

ComfyUI のワークフローが定期的またはトリガー駆動で実行され、生成した画像を自動的に配信サーバへ送信・配置する。運用者は ComfyUI のワークフローを設定するだけで、画像生成から配信までのパイプラインが自動化される。

**Why this priority**: 手動で画像を用意する手間を省き、システム全体の自動化を実現する核心機能。

**Independent Test**: ComfyUI ワークフローを実行後、配信サーバから画像が取得可能になっていることを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** ComfyUI ワークフローが設定済みの状態で、 **When** ワークフローが実行されたとき、 **Then** 生成された画像が自動的に配信サーバへ送信され、公開状態になる。
2. **Given** 配信サーバに新しい画像が配置されたとき、 **When** デバイスが次の更新サイクルで問い合わせたとき、 **Then** デバイスはその新しい画像を取得できる。
3. **Given** ComfyUI が生成した画像の解像度・フォーマットがデバイスの表示仕様と異なるとき、 **When** 画像が配信サーバに登録されるとき、 **Then** サーバは自動的にデバイス表示仕様に合わせた形式に変換する。

---

### User Story 3 - 配信サーバが複数デバイスへ画像を提供する (Priority: P3)

配信サーバが複数の PhotoPainter デバイスからのリクエストを処理し、適切な画像を提供する。デバイスごとに異なる画像を配信したり、共通の最新画像を一括配信したりできる。

**Why this priority**: 単一デバイス運用で価値は成立するが、複数台運用への拡張性を確保することでシステムの実用性が高まる。

**Independent Test**: 2 台以上のデバイスエミュレーターから同時にリクエストを送信し、それぞれが正常に画像を受信できることを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 複数の PhotoPainter デバイスが同じ WiFi ネットワークに存在するとき、 **When** 各デバイスが配信サーバへ画像を要求したとき、 **Then** すべてのデバイスが正常に画像を受信できる。
2. **Given** 配信サーバが稼働中のとき、 **When** デバイスが識別情報（デバイス ID など）を含めてリクエストしたとき、 **Then** サーバはそのデバイス向けの画像またはデフォルト画像を返す。

---

### Edge Cases

- ネットワーク接続が不安定でダウンロードが途中で中断された場合、デバイスは破損した画像を表示しない。
- 配信サーバに画像がまだ存在しない場合（初期状態）、デバイスは起動時にデフォルト表示またはプレースホルダーを維持する。
- ComfyUI が生成した画像のフォーマット・解像度がデバイスの表示仕様に合わない場合、配信サーバ側で自動変換する。
- WiFi 接続が切断された状態でデバイスが起動した場合、デバイスは WiFi 再接続を試みつつ、最後に表示した画像を維持する。
- 配信サーバが長時間停止していた後に再起動した場合、デバイスは次の更新サイクルで自動的に画像を取得できる。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: PhotoPainter デバイスのファームウェアは、WiFi 経由で配信サーバへ HTTP リクエストを送信し、画像データを取得できなければならない。
- **FR-002**: デバイスは設定可能な更新間隔（例: 毎時、毎日）で自動的に画像更新を試みなければならない。
- **FR-003**: デバイスはダウンロードした画像を e-ink ディスプレイに正しく表示できなければならない。
- **FR-004**: デバイスはネットワーク接続失敗やサーバ応答エラー時にも安定して動作し、現在の表示を維持しなければならない。
- **FR-005**: 配信サーバは画像ファイルを格納し、デバイスからの HTTP リクエストに対して画像を配信できなければならない。
- **FR-006**: 配信サーバは PhotoPainter の表示仕様（解像度・色深度）に合わせた形式で画像を提供しなければならない。
- **FR-007**: 配信サーバは ComfyUI から新しい画像を受け取り（プッシュまたはファイル配置）、最新の画像として管理できなければならない。
- **FR-008**: ComfyUI ワークフローは画像生成後、自動的に生成画像を配信サーバへ送信またはエクスポートできなければならない。
- **FR-009**: システム全体がローカルネットワーク（LAN/WiFi）内で自律的に動作し、外部インターネット接続なしでも機能しなければならない。
- **FR-010**: デバイス設定（サーバ URL、更新間隔など）は、ファームウェアの再書き込みなしに変更できなければならない。

### Key Entities

- **PhotoPainter デバイス**: WiFi 接続機能を持つ e-ink ディスプレイデバイス。デバイス ID、サーバ URL、更新間隔、最終更新日時などの属性を持つ。
- **配信サーバ**: 画像ファイルを管理・提供する HTTP サーバ。現在配信中の画像、デバイスごとの割り当てを管理する。
- **画像（Image）**: ComfyUI が生成または手動で登録した表示用画像。デバイス表示仕様に変換済みのフォーマット、生成日時、ソースメタデータを持つ。
- **ComfyUI ワークフロー**: 画像を自動生成して配信サーバへ送る自動化パイプライン。実行スケジュール、生成パラメータ、出力先設定を含む。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: デバイスへの画像更新が、設定した更新間隔の ±10% 以内のタイミングで自動実行される。
- **SC-002**: 安定したネットワーク環境下で、デバイスが画像を取得・表示する処理を 60 秒以内に完了できる。
- **SC-003**: ネットワーク障害からの回復後、デバイスが次の更新サイクルで自動的に画像を取得できる（手動介入不要）。
- **SC-004**: ComfyUI でワークフローを実行後、生成された画像が 5 分以内にデバイスで取得可能な状態になる。
- **SC-005**: 少なくとも 5 台の PhotoPainter デバイスを同一サーバで問題なく運用できる。
- **SC-006**: デバイスの設定（サーバ URL、更新間隔）をファームウェア再書き込みなしに変更できる。

## Assumptions

- PhotoPainter デバイスは Waveshare 製などの e-ink ディスプレイデバイスであり、WiFi 接続可能なマイコン（ESP32 等）を内蔵している前提。
- 配信サーバとデバイスは同一ローカルネットワーク（WiFi）に存在する前提。外部公開は初期スコープ外。
- ComfyUI はサーバと同一マシンまたはローカルネットワーク上のマシンで稼働している前提。
- 画像の更新頻度は数時間〜数日単位であり、リアルタイム性は不要。
- 配信する画像は 1 デバイスにつき「現在表示すべき最新画像 1 枚」の管理を基本とし、複雑なコンテンツスケジューリングは初期スコープ外。
- デバイスの初期 WiFi 設定（SSID/パスワード、サーバ URL）は初回ファームウェア書き込み時または専用設定モードで行う前提。
- 表示画像の著作権・商用利用については運用者の責任とし、システム側では管理しない。
