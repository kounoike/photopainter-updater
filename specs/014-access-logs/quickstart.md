# Quickstart: HTTP アクセスログ追加

## 1. 正常系アクセスログ確認

1. `server/contents/image.png` を配置する。
2. `server/run.sh` でサーバを起動する。
3. `GET /` にアクセスする。
4. `GET /image.bmp` にアクセスする。
5. 各アクセスごとに 1 行ずつアクセスログが出力され、path と `200` が確認できることを確認する。

## 2. 失敗系アクセスログ確認

1. `image.png` を未配置にするか、変換不能な内容へ差し替える。
2. `GET /` または `GET /image.bmp` にアクセスする。
3. アクセスログが出力され、失敗ステータスが確認できることを確認する。

## 3. 未定義 path のアクセスログ確認

1. サーバ起動中に存在しない path へアクセスする。
2. 通常の失敗応答を維持したまま、アクセスログにも対象 path と失敗ステータスが残ることを確認する。

## 4. 複数回アクセス確認

1. 同じ path へ複数回連続でアクセスする。
2. 各アクセスが個別のログ行として区別して確認できることを確認する。
3. 取得可能な場合は、各ログ行にアクセス元情報が含まれることを確認する。

## 5. 既存レスポンス契約維持確認

1. `GET /` と `GET /image.bmp` のレスポンスを取得する。
2. `image/bmp` の Content-Type と従来どおりの成功・失敗ステータスが維持されていることを確認する。
3. アクセスログ追加前提でも、レスポンス body や失敗応答の意味が変わっていないことを確認する。
