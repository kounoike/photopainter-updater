# Quickstart: Devcontainer 起動改善

## 1. 新規作成直後の利用確認

1. devcontainer を新規 build/create する。
2. 接続完了後、`codex` と `claude` を起動する。
3. 追加の手動インストールを実施せずに両 CLI が利用可能であることを確認する。
4. `docs/firmware.md` に記載した手順どおり、CLI が image build 時点で同梱されていることを確認する。

## 2. 初回認証と再利用確認

1. 永続領域が空の状態で devcontainer に接続する。
2. `codex` と `claude` を通常手順で認証する。
3. devcontainer を rebuild または recreate する。
4. 再接続後、`codex` と `claude` を起動する。
5. 両方とも再認証を求められず継続利用できることを確認する。
6. ワークスペース配下に認証ファイルが新規作成されていないことを確認する。

## 3. Git 管理対象からの分離確認

1. 認証完了後にワークスペースの変更状態を確認する。
2. 認証関連ファイルがリポジトリの追跡対象として現れないことを確認する。

## 4. 初期化手順の確認

1. 開発者向け手順書を開く。
2. `codex` と `claude` の認証キャッシュ保持対象と、永続領域の初期化方法を確認する。
3. 必要であれば文書化された手順で永続領域を初期化し、再度初回認証手順へ戻れることを確認する。
4. 初期化方法が host 側の named volume 削除であり、リポジトリ配下のファイル削除を要求していないことを確認する。
