# Quickstart: run.sh 配信設定改善

## 1. 既定ディレクトリで起動する

1. `server/contents/` に配信したい内容を置く。
2. `server/run.sh` を起動する。
3. 起動案内に `Local:` と `LAN:` の両方が表示され、別端末利用方法が localhost 専用ではないことを確認する。
4. 同一ホストから配信内容へアクセスできることを確認する。

## 2. LAN 端末から到達できることを確認する

1. サーバを起動したホストの LAN アドレスを確認する。
2. 同一ネットワーク上の別端末から、その LAN アドレスと表示ポートを使って配信 URL へアクセスする。
3. 配信内容を 1 回のアクセスで取得できることを確認する。

## 3. 任意ディレクトリを配信元にして起動する

1. 任意のディレクトリに配信したい内容を置く。
2. そのディレクトリを指定して `server/run.sh` を起動する。
3. 起動案内に表示された配信元が、指定ディレクトリ配下の `image.bmp` を指していることを確認する。
4. 配信結果が指定ディレクトリ配下の `image.bmp` と一致することを確認する。

## 4. 別ディレクトリから起動する

1. リポジトリ外または別の作業ディレクトリへ移動する。
2. 絶対パスまたは相対参照で `server/run.sh` を起動する。
3. 起動案内に表示される配信元が起動元カレントディレクトリではなく、解決後の実ディレクトリになっていることを確認する。
4. 既定ディレクトリ利用時も任意ディレクトリ指定時も、配信結果が期待どおりであることを確認する。

## 5. 異常系を確認する

1. 存在しない配信元ディレクトリを指定して `server/run.sh` を起動する。
2. サーバが起動せず、原因が配信元ディレクトリ不正だと判別できることを確認する。
3. `image.bmp` を含まない空の配信元ディレクトリを指定して `server/run.sh` を起動する。
4. サーバは起動し、アクセス時に `image.bmp` 未配置または内容不足だと判別できることを確認する。
